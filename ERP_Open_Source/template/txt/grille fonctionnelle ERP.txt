3- Caracteristiques techniques
       - type d'architecture : client/serveur, Web, autre
       - base de donnees : MySQL, Oracle, PostgreSQL, etc.
       - Interopérabilité
       - Portabilité
       - Multilangue (1 seule langue / Internationalisation possible, quelques langues dispo / disponible dans de nombreuses langues)
       - Migration à partir d'autres solutions
       - respect des standards
       
       - Installation
       		- facilité (Installation automatique 	
		- Configuration ( longue et fastidieuse / indispensable mais pas trop lourde /automatique, outils fonctionnel clef en main )
	- Utilisation
		- prise en main ( très lentes, plusieurs jours / moyennne quelques heures / rapide (-1 heure)
		- Ergonomie ( on ne s'y retrouve pas sans mode d'emploi / correct / très intuitif, quasi aucune doc nécessaire)
		- qualité de l'IHM (pas beau, fouilli/ correct / clair, beau, plusieurs thèmes)
		
	- Qualité du code source
		- Code source bien structuré, lisible
		- Absence de langage propriétaire
		- Absence de brique propriétaire (Ex : Compiere uniquement sur Oracle)
		- Développement de plugins possible
		- Navigation claire dans les sources (arborescence)
		
	- Qualité du support/ de la communauté
		- Support
		- taille de la communauté
		- Internationalisme de la communauté
		- perenité
		- forum de support (aucun / présent mais peu de participation ou peu de réactivité /Bonne réactivité sur le forum, participation en masse )
		- wiki de support (aucun / présent mais peu d'article / présent et bien fourni )

	- Documentation 
		- disponibilité ( payante / papier uniquement / en ligne gratuit )
		- multilangue ( 1 seule langue anglais / 1 seule langue français / plusieurs )
		- tutoriaux ( aucun / quelques uns basiques / nombreux et avancés)
		- Caractéristiques en lignes 	
		- Version(s) de démo en ligne ( aucune / 1 peu représentative / plusieurs et très parlante)
		- FAQ (absente / présente mais peu fournie / présente et très complète)

	- Exploitation
		- montée en charge ( catastrophique, performances qui s'effondrent / correcte / excellente )
		- Utilisation des ressources serveur ( mauvaise / correcte / excellente )
		- temps de réponse ( très lent / correcte / très rapide )
		- disponibilité
		- stabilité
		- Facilité de mise en production
